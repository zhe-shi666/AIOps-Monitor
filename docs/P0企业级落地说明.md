# P0 企业级落地说明

本轮 P0 目标是把系统从“可演示”推进到“可生产试运行”的基础版。

## 已落地能力

1. 目标级阈值策略：每个监控目标可以覆盖用户默认阈值。
2. Agent 服务化安装：提供 Linux systemd、macOS launchd、Windows PowerShell 服务化入口。
3. Agent 版本治理：后端提供最新 Agent 版本，前端能识别过期 Agent。
4. 生产级角色基础：支持 ADMIN、OPS、AUDITOR、USER，写操作限制为 ADMIN/OPS。
5. 审计日志：关键目标操作、阈值修改、Key 轮换会写入审计日志，并提供“运维审计”前端页面分页检索。
6. 数据保留策略：指标、日志、Trace、已解决事件、异常、审计日志按周期清理。
7. 数据库迁移治理：新增 P0 独立 SQL 迁移脚本 `mysql/migrations/2026-04-30_p0_enterprise_foundation.sql`。

## 当前边界

- 角色权限已经具备后端基础拦截；监控目标页已对只读角色禁用写操作入口，后续可继续扩展到按钮级权限矩阵。
- 数据库迁移仍是 SQL 脚本执行方式，尚未接入 Flyway/Liquibase 自动迁移。
- Windows 服务化当前使用 PowerShell 进程管理基础版，生产环境后续可升级为 NSSM 或原生 Windows Service。
