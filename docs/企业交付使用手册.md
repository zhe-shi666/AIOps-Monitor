# AIOps Monitor 企业交付使用手册

本文面向企业客户、交付工程师和售后运维人员，说明系统如何落地、如何接入主机、如何处理告警、如何使用 AI 专家。

## 1. 系统定位

AIOps Monitor 是面向中小企业和私有化环境的智能监控平台。核心目标不是只看 CPU 和内存，而是把主机指标、日志、Trace、告警、AI 调查和人工审核串成闭环。

平台当前覆盖：

- 主机指标：CPU、内存、磁盘、网络 RX/TX、进程数。
- 主机状态：在线、离线、禁用、最后心跳、Agent 版本、真实 IP。
- 告警事件：阈值触发、连续触发、静默窗口、重复事件降噪。
- AI 专家：从事件工作台发起调查，生成结构化分析、行动建议、报告草稿和快照。
- 证据源：Agent 周期性上报指标、心跳日志、采集 Trace。
- 多用户隔离：目标、事件、调查、通知策略按登录用户隔离。

## 2. 标准操作流程

### 2.1 添加监控目标

1. 登录系统。
2. 进入「监控目标」。
3. 点击「新增目标」。
4. 填写目标名称、主机名、描述。
5. 创建后系统生成唯一 Agent Key。
6. 在右侧「安装接入」中选择目标系统：Linux、macOS 或 Windows。
7. 复制对应系统的安装/升级命令，并在被监控主机执行。
8. 回到目标列表，确认状态变为 `ONLINE`。

### 2.2 Agent 安装后的验证

Linux/macOS：

```bash
cd ~/aiops-agent-lite
./status.sh
./tail-log.sh
```

Windows PowerShell：

```powershell
Set-Location "$HOME\aiops-agent-lite"
powershell -ExecutionPolicy Bypass -File .\status.ps1
powershell -ExecutionPolicy Bypass -File .\tail-log.ps1
```

成功日志应包含：

```text
metricMode: 'real-cross-platform'
observabilityEnabled: true
[register] ... ONLINE
[heartbeat] ... DISK=... RX=... TX=... PROC=... true
```

## 3. 日常运维流程

### 3.1 告警处理

1. 进入「事件工作台」。
2. 左侧选择告警。
3. 右侧查看告警摘要、级别、主机、状态。
4. 点击「加载证据」查看附近日志和 Trace。
5. 如果确认需要诊断，点击「发起调查」。
6. 系统跳转「AI 专家」。

### 3.2 AI 专家流程

1. 在 AI 专家页面选择调查对象。
2. 点击「AI 结构化分析」。
3. 查看根因假设、证据、风险等级、行动建议。
4. 点击「生成草稿」生成复盘/报告。
5. 人工审核内容。
6. 保存快照。
7. 需要执行动作时，先审批，再执行，必要时回滚。

AI 专家价值：

- 把告警翻译成可理解的诊断结论。
- 自动组织证据，减少人工翻日志时间。
- 输出结构化根因假设和操作建议。
- 支持人工审核，避免 AI 自动误操作。
- 保存报告快照，便于审计和交付复盘。

## 4. 离线目标处理

如果 Agent 停止或网络断开：

- 目标会变为 `OFFLINE`。
- 前端目标卡片会置灰。
- 最后一批指标保留，用于回看。
- 曲线不应继续增长，避免误判为实时数据。

排查命令：

```bash
cd ~/aiops-agent-lite
cat .env
./status.sh
./tail-log.sh
curl -i "$API_BASE_URL/actuator/health"
```

常见原因：

- `API_BASE_URL` 写错。
- `AGENT_KEY` 已轮换但目标机器未更新。
- 后端 8080 未启动。
- Mac/服务器防火墙阻断。
- 目标被禁用。

## 5. 告警规则策略

当前系统支持用户级阈值策略：

- CPU 阈值。
- 内存阈值。
- 磁盘阈值。
- 进程数阈值。
- 连续超阈值次数。
- 静默窗口。

企业推荐配置：

- CPU：85%。
- 内存：90%。
- 磁盘：85%。
- 进程数：按业务基线设置。
- 连续触发：2 到 3 次。
- 静默窗口：180 到 600 秒。

说明：多目标环境下先使用统一基线，后续如企业需要精细化，可以扩展为目标级策略。

## 6. 通知与升级

通知通道用于把告警推送到企业内部系统，例如飞书、钉钉、企业微信、Webhook 网关。

标准流程：

1. 进入「通知通道」。
2. 新增 Webhook。
3. 配置密钥。
4. 进入「升级策略」。
5. 配置 P1/P2/P3 的通知节奏。
6. 在事件工作台查看投递记录。

## 7. 交付验收清单

- 可以登录注册。
- 可以新增监控目标。
- Linux、macOS、Windows Agent 可以安装并上报真实指标。
- 停止 Agent 后目标会离线。
- CPU/内存/磁盘/网络/进程数都有数据。
- 告警能进入事件工作台。
- 重复告警能折叠展示。
- 事件能发起 AI 调查。
- AI 专家能生成结构化分析和报告草稿。
- 快照能保存。
- 多用户之间目标和事件互不可见。
- 通知投递有审计记录。

## 9. 企业卖点

- 私有化部署，适合企业内网。
- 多主机 Agent 接入，不要求目标机器部署完整项目。
- 指标、日志、Trace 三类证据统一进入 AI 调查。
- 告警降噪和静默窗口降低重复打扰。
- AI 专家不直接替人执行，而是提供人工可审核的诊断和操作建议。
- 支持快照、审计、通知投递记录，适合企业交付。
