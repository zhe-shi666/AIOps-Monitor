# Phase-5 交付与验收说明

更新时间：2026-04-28

## 1. 交付范围

1. 设计系统收敛（核心页面统一）  
- 统一 `page-surface / page-hero / card-panel / kpi-grid / ops-table` 组件化样式体系。  
- 顶部主题/语言控制、导航和页面标题系统统一生效。  
- 新增事件工作台页面并纳入同一套视觉规范。

2. 事件工作台闭环  
- 新增页面：`/workbench`（Event Workbench）。  
- 同屏支持：告警筛选、状态处理、投递审计、发起调查、跳转 AI 专家执行。  
- 支持按 `investigationId` 深链进入 AI 专家并自动选中调查。

3. 前端性能收口  
- ECharts 改为按需动态加载（`echarts/core` + runtime import）。  
- Vite 手工分包策略：`vendor-vue / vendor-element / vendor-echarts / vendor-realtime / vendor-misc`。  
- 提升首次可用性，降低核心路由首屏阻塞风险。

## 3. Phase-5 Exit Criteria 映射

1. 设计系统在核心页面统一生效  
- 已完成：总览、工作台、AI 专家、目标、告警、设置页复用统一样式体系与导航/主题机制。

2. 事件工作台支持全流程闭环  
- 已完成：告警处理 → 调查创建/打开 → AI 专家执行入口打通。

3. 前端性能满足目标（关键页面可用）  
- 已完成：核心重模块（图表）延迟加载，Vendor 分包策略生效，构建通过。

## 4. 验证方式

1. 登录后进入 `/workbench`，完成以下链路：  
- 修改告警状态（OPEN/ACKNOWLEDGED/RESOLVED）  
- 触发“发起调查”或“打开调查”  
- 跳转 `/ai-expert?investigationId=...` 并自动定位调查

2. 构建验证：  
- `cd aiops-vue && npm run build`
