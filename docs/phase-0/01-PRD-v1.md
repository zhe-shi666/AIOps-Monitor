# AIOps-Monitor PRD（V1 冻结版）

> 版本：v1  
> 日期：2026-04-27  
> 产品定位：面向多租户企业用户的智能运维监控平台

## 1. 背景与目标

当前项目已具备 CPU/内存基础监控、WebSocket 实时展示、AI 报告雏形。下一阶段目标不是做演示型 MVP，而是落地可持续演进的企业级平台。

### 1.1 业务目标

1. 支持多租户、可扩展的主机监控平台。
2. 将“监控”升级为“发现异常 -> 关联分析 -> 诊断建议 -> 处置闭环”。
3. 在不牺牲后端简洁性的前提下，构建高级、专业的前端运维体验。

### 1.2 技术目标

1. 统一监控数据模型（主机、中间件、应用、日志、事件）。
2. 分阶段交付，阶段冻结，不回头返工。
3. 保证可观测、可审计、可扩容、可回滚。

## 2. 产品范围（V1）

### 2.1 In Scope（纳入范围）

1. 资产与采集
- 主机资产管理（虚机/物理机）
- Agent 注册与鉴权
- CPU、内存、磁盘、文件系统、网络、进程指标采集

2. 监控与告警
- 指标存储与查询
- 规则告警（阈值、持续时长、组合条件）
- 告警聚合、去重、抑制、静默

3. 事件与 AI
- 事件中心（告警、变更、系统事件）
- 异常检测（动态基线）
- 根因候选分析（基于时序 + 拓扑 + 变更）
- AI 诊断报告与建议

4. 平台能力
- 多租户逻辑隔离
- RBAC 权限体系
- 操作审计日志
- 可配置通知通道（企业微信/邮件/Webhook）

5. 前端体验
- 统一设计系统（字体、色板、组件、动效）
- 实时大盘、事件工作台、诊断详情页
- 租户维度视图与资产维度视图

### 2.2 Out of Scope（暂不纳入）

1. 全量 SIEM 安全事件平台能力。
2. 全自动闭环修复（无审批直接执行高危动作）。
3. 跨云 CMDB 深度自动发现（仅保留接口与扩展点）。

## 3. 用户与角色

1. 平台管理员（Platform Admin）
- 管理租户、套餐、全局策略、系统配置

2. 租户管理员（Tenant Admin）
- 管理本租户资产、用户、告警策略、通知规则

3. 运维工程师（Operator）
- 日常监控、告警处理、事件排查、执行 Runbook

4. 只读用户（Viewer）
- 查看仪表盘、报表与事件结果，无变更权限

## 4. 核心业务流程

1. Agent 注册主机 -> 通过鉴权并绑定租户。
2. 指标/日志/事件上报 -> 统一处理与存储。
3. 规则引擎 + 异常检测并行输出告警。
4. 事件中心聚合告警并做降噪。
5. AI 引擎生成 RCA 候选与修复建议。
6. 运维人员执行 Runbook（可审批）并形成审计记录。
7. 事件关闭后进入复盘与知识库沉淀。

## 5. 非功能需求

1. 可用性
- 控制面 SLA 目标：99.9%
- 采集链路可用性目标：99.95%

2. 性能
- P1 告警触发到通知：<= 30 秒
- 大盘刷新延迟：<= 5 秒（常规场景）

3. 安全
- 全链路鉴权
- 角色最小权限
- 审计日志不可篡改（逻辑层）

4. 可运维
- 支持健康检查、指标暴露、链路追踪
- 明确回滚策略与故障演练机制

## 6. 默认容量基线（通用行业模板）

1. 默认租户配额：100 台主机，可升配。
2. 目标容量分层
- L1：1,000 台主机
- L2：10,000 台主机
- L3：50,000 台主机
3. 指标采样
- 主机核心指标：15s
- 中间件：30s
- 业务指标：30s~60s
4. 保留策略
- 指标热数据：15 天
- 指标温数据：90 天
- 指标归档：365 天
- 日志在线检索：30 天
- 日志归档：180 天

## 7. 验收指标（V1）

1. 功能验收
- 主机监控覆盖率 >= 95%
- 告警聚合去重可用
- AI 报告可追溯到源事件

2. 体验验收
- 关键页面加载与交互稳定
- 视觉规范一致（组件与主题统一）

3. 质量验收
- 关键流程有自动化测试
- 关键接口有监控与审计

## 8. 范围冻结与变更机制

1. 本文档冻结后，新增大需求默认进入下一阶段 Backlog。
2. 当前阶段仅接收 P0（线上阻断）和 P1（安全合规）变更。
3. 需求变更必须提交：变更原因、影响范围、风险、回滚方式。
4. 未完成 DoD 不得进入下一阶段开发。
