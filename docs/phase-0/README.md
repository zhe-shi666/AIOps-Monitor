# Phase 0 冻结文档（V1）

> 冻结日期：2026-04-27  
> 适用路线：虚机/物理机优先（兼容未来 K8s 扩展）

本目录用于固化项目启动阶段的共识，避免“边做边改方向”。

## 文档清单

1. `01-PRD-v1.md`：产品需求与范围冻结
2. `02-总体架构设计-v1.md`：系统架构、模块边界、部署与演进
3. `03-数据字典-v1.md`：核心领域模型、统一字段规范、指标标签规范
4. `04-里程碑与DoD-v1.md`：阶段计划、验收标准、GitHub Desktop 执行规范

## 执行规则

1. 先按本目录冻结版本执行，不跨阶段插入大需求。
2. 需求变更使用“变更单”机制，默认进入下一阶段。
3. 每阶段都必须满足 DoD 才能进入下一阶段。
4. 文档版本采用 `v1 -> v1.1 -> v2`，所有变更保留历史记录。
