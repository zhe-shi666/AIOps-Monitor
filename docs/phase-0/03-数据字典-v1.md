# AIOps-Monitor 数据字典（V1 冻结版）

> 版本：v1  
> 日期：2026-04-27

## 1. 命名与通用字段规范

1. 主键统一：`id`（bigint 或 uuid，按模块统一）。
2. 多租户必带：`tenant_id`。
3. 审计字段统一：`created_at`、`updated_at`、`created_by`、`updated_by`。
4. 软删除统一：`is_deleted`（bool）。
5. 状态字段统一：`status`（枚举字符串）。

## 2. 核心实体

## 3. 主机指标最小集合（V1）

1. CPU
- `host.cpu.usage.percent`
- `host.cpu.load.1m`

2. 内存
- `host.memory.used.percent`
- `host.memory.available.bytes`

3. 磁盘与文件系统
- `host.disk.used.percent`
- `host.disk.read.bytes_per_sec`
- `host.disk.write.bytes_per_sec`

4. 网络
- `host.net.rx.bytes_per_sec`
- `host.net.tx.bytes_per_sec`
- `host.net.error.count`

5. 进程
- `host.process.count`
- `host.process.cpu.topn`
- `host.process.mem.topn`

## 4. 标签（Labels）规范

1. 必选标签
- `tenant`
- `host`
- `env`（prod/stage/dev）
- `region`（可为空）

2. 可选标签
- `service`
- `app`
- `team`
- `idc`

3. 约束
- 标签 key 使用小写蛇形或点分命名。
- 禁止高基数字段直接作为标签（如 request_id、user_id）。

## 5. 告警事件状态机

1. `open`：首次触发。
2. `acknowledged`：人工确认处理中。
3. `suppressed`：被抑制策略命中。
4. `resolved`：条件恢复。
5. `closed`：事件关闭归档。

## 6. 数据保留与归档（默认）

1. metric_sample：热 15 天，温 90 天，归档 365 天。
2. log_record：在线 30 天，归档 180 天。
3. incident/audit：建议保留 >= 365 天。

## 7. 数据字典冻结规则

1. 新增字段允许，删除或语义变更必须走变更评审。
2. 字段命名冲突以本字典为准。
3. API 与 DB 结构必须引用同一字段定义，禁止同义多名。
