# 企业安全与迁移说明

本文档解释本轮企业化收口新增的安全、迁移、Agent 发布与权限控制能力。

## 1. 自动数据库迁移

系统已接入 Flyway。客户部署时只需要启动后端，不再手工执行 SQL。

迁移目录：

```text
ai-ops-monitor/src/main/resources/db/migration
```

当前版本：

- `V1__baseline_schema.sql`：当前企业版基础表结构。
- `V2__enterprise_security_and_agent_release.sql`：密码重置令牌、Agent 发布表、角色枚举扩展。
- 老库接管：`baseline-version=1`，已有业务表的数据库会跳过 V1，只执行 V2 之后的增量迁移。
- 新库部署：空数据库会正常执行 V1 创建基础表。

开发规则：

- 已发布的 `V*.sql` 不允许修改。
- 所有后续变更必须新增版本号，例如 `V3__add_xxx.sql`。
- 本地和生产都由 Flyway 自动执行。

## 2. 登录账号安全

已实现：

- 密码强度校验：长度、大小写、数字、特殊字符。
- 公开注册开关：生产建议关闭。
- 初始管理员自动初始化：仅用户表为空时创建。
- 邮箱找回密码：用户输入用户名和绑定邮箱，匹配后发送重置链接。
- 密码重置令牌：默认 30 分钟有效，存储 SHA-256 哈希，使用后失效。
- JWT Secret 环境变量化。

生产建议：

```bash
REGISTRATION_ENABLED=false
BOOTSTRAP_ADMIN_ENABLED=true
JWT_SECRET=$(openssl rand -base64 64)
```

密码重置邮件配置：

```bash
PASSWORD_RESET_MAIL_ENABLED=true
PASSWORD_RESET_FRONTEND_URL=https://你的域名/reset-password
PASSWORD_RESET_TOKEN_TTL_MINUTES=30
MAIL_HOST=smtp.example.com
MAIL_PORT=587
MAIL_USERNAME=no-reply@example.com
MAIL_PASSWORD=SMTP授权码或密码
MAIL_SMTP_AUTH=true
MAIL_SMTP_STARTTLS_ENABLE=true
PASSWORD_RESET_MAIL_FROM=no-reply@example.com
```

流程：

1. 用户在登录页点击“忘记密码”。
2. 输入用户名和绑定邮箱。
3. 后端校验用户名和邮箱是否属于同一账号。
4. 匹配成功后发送邮件，邮件链接指向 `/reset-password?token=...`。
5. 用户设置新密码后，token 标记为已使用。

## 3. Agent 安装包签名与升级

已实现：

- Agent 打包脚本生成 tar.gz。
- 自动计算 SHA256。
- 使用 `AGENT_SIGNING_KEY` 生成 HMAC-SHA256 签名。
- 生成 `agent-release.json` 与 `agent-release.sql`。
- 安装/升级脚本校验 SHA256 与签名后再替换本地 Agent。
- 升级时保留目标机器 `.env` 配置。

发布命令：

```bash
AGENT_SIGNING_KEY='生产签名密钥' AGENT_VERSION='agent-lite-1.2.1' ./tools/agent-lite/package-agent-lite.sh
```

## 4. 前后端权限矩阵

角色定义：

| 角色 | 能力 |
| --- | --- |
| ADMIN | 全部权限、用户管理、配置变更、审计查看 |
| OPS | 运维操作、告警处理、AI 调查、配置变更、审计查看 |
| AUDITOR | 审计与只读查看，不能写操作 |
| USER | 普通只读用户，不能写操作 |

前端已实现：

- 路由级权限拦截。
- 导航菜单按角色显示。
- 设置页、事件页、AI 专家页按钮级只读保护。
- 只读角色显示禁用原因。

后端已实现：

- 管理后台接口 ADMIN 限制。
- 阈值、通知、升级策略、事件状态、AI 调查写操作要求 ADMIN/OPS。
- 读接口继续按当前用户隔离数据。
