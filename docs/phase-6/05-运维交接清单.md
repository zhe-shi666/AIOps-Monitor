# 运维交接清单（上线前/上线后）

## 1. 上线前检查

1. 代码版本  
- 发布分支与 commit 已冻结  
- 回滚版本号已确认

2. 配置与依赖  
- MySQL/Redis/Ollama/Prometheus 运行正常  
- 关键环境变量已配置（JWT、OLLAMA_BASE_URL、OLLAMA_MODEL）

3. 发布门禁  
- `./mvnw -q test` 通过  
- `npm run build` 通过  
- 压测门禁通过  
- 灰度门禁通过

4. 人员安排  
- 发布负责人  
- 回滚负责人  
- 值班 SRE  
- 业务通知人

## 2. 上线后 30 分钟观察项

1. 服务健康  
- `/actuator/health` 为 UP  
- CPU/内存趋势稳定

2. 业务可用  
- 登录可用  
- 告警中心可读写  
- AI 专家页面可加载调查并执行流程

3. 告警质量  
- 错误日志无异常激增  
- 告警噪声无突增

## 3. 交接输出物

1. 发布记录（时间、版本、负责人）  
2. 压测报告（参数 + 结果）  
3. 演练记录（场景 + 恢复耗时）  
4. 灰度记录（每阶段观察结论）  
5. 回滚预案确认（触发条件 + 命令 + 负责人）

## 4. 值班约定

1. 发布后 24 小时内进入增强值守。  
2. 如触发 P1，5 分钟内启动应急群并执行回滚判定。  
3. 发布后 48 小时内输出复盘（即使无故障，也输出观测总结）。
