# 压测与容量基线

## 1. 目标

验证关键读链路在目标并发下满足可用性要求：

1. 错误率 <= 5%  
2. P95 <= 1200ms  
3. 核心接口可持续响应

## 2. 执行命令

```bash
cd /path/to/AIOps-Monitor

API_BASE_URL=http://localhost:8080 \
USERNAME=admin PASSWORD=123456 AUTO_REGISTER=true \
CONCURRENCY=20 DURATION_SECONDS=60 \
MAX_ERROR_RATE=0.05 MAX_P95_MS=1200 \
node tools/phase-6/load/http-load.mjs
```

## 3. 推荐分层基线（中小企业）

1. L1（入门）  
- 并发：10  
- 时长：60s  
- 目标：错误率 <= 3%，P95 <= 900ms

2. L2（标准）  
- 并发：20  
- 时长：120s  
- 目标：错误率 <= 5%，P95 <= 1200ms

3. L3（增强）  
- 并发：40  
- 时长：180s  
- 目标：错误率 <= 8%，P95 <= 1800ms

## 4. 结果判定

1. 通过：脚本返回 `gate passed`。  
2. 失败：脚本返回 `gate failed`，应暂停发布并做以下排查：
- 数据库连接池/慢 SQL  
- WebSocket 推送与 HTTP 查询竞争  
- AI 推理超时对主链路的影响  
- 容器资源限制（CPU/Memory）

## 5. 压测后动作

1. 保存压测输出 JSON 到发布记录。  
2. 记录当时版本号、环境、并发参数。  
3. 若失败，进入缺陷修复分支，不得直接灰度。
