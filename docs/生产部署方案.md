# AIOps Monitor 生产部署方案

本文档用于企业客户单机/虚机部署 AIOps Monitor。目标是让客户不再手工执行 SQL，不遗漏环境变量，并能通过 HTTPS 对外提供前端、后端 API、WebSocket 与 Agent 安装包。

## 1. 部署架构

```text
用户浏览器/Agent
        |
      HTTPS
        |
      Nginx
   |     |      |
前端   后端API  /agent静态包
        |
   MySQL + Redis + Ollama
```

- Nginx：统一入口，负责 HTTPS、前端静态资源、API 反代、WebSocket 反代、Agent 包下载。
- Frontend：Vue 构建后的静态页面。
- Backend：Spring Boot 服务，启动时通过 Flyway 自动迁移数据库。
- MySQL：业务数据、指标历史、AI 调查、审计日志。
- Redis：实时消息与分布式广播。
- Ollama：本地大模型推理服务。

## 2. 服务器要求

- CPU：4 核起步，推荐 8 核。
- 内存：8 GB 起步，推荐 16 GB 以上；如果 Ollama 使用更大模型，需要额外增加内存。
- 磁盘：100 GB 起步，生产建议 500 GB 以上，并使用独立数据盘保存 Docker volume 与备份。
- 系统：Linux x86_64，推荐 Ubuntu 22.04/24.04 或 Rocky Linux 9。
- 软件：Docker 24+，Docker Compose v2。

## 3. 首次部署

1. 准备生产环境变量：

```bash
cp .env.production.example .env.production
nano .env.production
```

2. 必须修改以下变量：

```bash
MYSQL_ROOT_PASSWORD=强随机 root 密码
MYSQL_PASSWORD=强随机业务库密码
REDIS_PASSWORD=强随机 Redis 密码
JWT_SECRET=至少 64 位强随机字符串
BOOTSTRAP_ADMIN_PASSWORD=符合密码策略的初始管理员密码
AGENT_SIGNING_KEY=强随机 Agent 发布签名密钥
VITE_API_BASE_URL=https://你的域名
VITE_WS_BASE_URL=https://你的域名
VITE_AGENT_BASE_URL=https://你的域名/agent
PASSWORD_RESET_MAIL_ENABLED=true
PASSWORD_RESET_FRONTEND_URL=https://你的域名/reset-password
MAIL_HOST=SMTP服务器
MAIL_USERNAME=发件邮箱
MAIL_PASSWORD=SMTP授权码或密码
```

3. 准备 HTTPS 证书：

```bash
mkdir -p deploy/certs
# 放入：deploy/certs/fullchain.pem
# 放入：deploy/certs/privkey.pem
```

4. 修改 Nginx 域名：

```bash
nano deploy/nginx/aiops.conf
# 将 server_name aiops.example.com; 改为客户真实域名
```

5. 生成 Agent 发布包：

```bash
AGENT_SIGNING_KEY='与 .env.production 相同的签名密钥' ./tools/agent-lite/package-agent-lite.sh
```

6. 启动生产服务：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml up -d --build
```

7. 拉取 Ollama 模型：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml exec ollama ollama pull qwen3:0.6b
```

8. 查看服务状态：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml ps
curl -k https://你的域名/actuator/health
```

## 4. 数据库自动迁移

后端已接入 Flyway：

- 迁移脚本目录：`ai-ops-monitor/src/main/resources/db/migration`
- 首次启动自动执行：`V1__baseline_schema.sql`、`V2__enterprise_security_and_agent_release.sql`
- 以后新增字段，只需要新增 `V3__xxx.sql`、`V4__xxx.sql`，禁止再让客户手工执行零散 SQL。

生产配置：

```yaml
spring.flyway.enabled: true
spring.flyway.locations: classpath:db/migration
spring.flyway.baseline-on-migrate: true
spring.flyway.baseline-version: 1
spring.jpa.hibernate.ddl-auto: validate
```

说明：

- 新库：Flyway 会自动建表和记录 schema history。
- 老库：`baseline-on-migrate=true` 与 `baseline-version=1` 用于接管已有库，把 `V1__baseline_schema.sql` 视为已存在的基线，只继续执行 V2 之后的增量脚本。
- 如果启动失败提示 migration checksum mismatch，不要删除业务表，先确认是否修改过已发布的 migration 文件。已发布 migration 不允许改内容，只能新增版本。

如果本地老库曾经错误执行过 V1 并留下失败记录，可以只清理 Flyway 元数据，不要动业务表：

```bash
docker exec aiops-mysql mysql -uroot -proot aiops_monitor -e "DELETE FROM flyway_schema_history WHERE success = 0; UPDATE flyway_schema_history SET version = '1', description = 'Flyway baseline for existing schema' WHERE type = 'BASELINE';"
```

## 5. 初始管理员与账号策略

生产建议：

```bash
REGISTRATION_ENABLED=false
BOOTSTRAP_ADMIN_ENABLED=true
BOOTSTRAP_ADMIN_USERNAME=admin
BOOTSTRAP_ADMIN_EMAIL=admin@example.com
BOOTSTRAP_ADMIN_PASSWORD=强密码
PASSWORD_MIN_LENGTH=10
```

系统启动时：

- 如果用户表为空，会自动创建初始管理员。
- 如果已有用户，不会重复创建管理员。
- 公开注册默认关闭，客户账号由管理员创建或后续后台管理。
- 密码策略默认要求：长度、大小写、数字、特殊字符。
- JWT Secret 必须使用生产强随机值，不允许使用示例值。

生成随机密钥示例：

```bash
openssl rand -base64 64
```

## 6. Agent 发布与升级

打包发布：

```bash
AGENT_SIGNING_KEY='生产签名密钥' AGENT_VERSION='agent-lite-1.2.1' ./tools/agent-lite/package-agent-lite.sh
```

产物：

- `dist/aiops-agent-lite.tar.gz`
- `dist/agent-release.json`
- `dist/install-or-upgrade.sh`
- `dist/agent-release.sql`

写入数据库发布信息：

```bash
docker exec -i aiops-mysql mysql -uroot -p$MYSQL_ROOT_PASSWORD aiops_monitor < dist/agent-release.sql
```

目标机器升级：

```bash
AGENT_SIGNING_KEY='生产签名密钥' AIOPS_AGENT_BASE_URL='https://你的域名/agent' \
  curl -fsSL https://你的域名/agent/install-or-upgrade.sh | bash
```

升级脚本会校验：

- `sha256`：确认包没有损坏。
- `signature`：确认包由同一签名密钥发布。
- `.env`：保留原机器的 `API_BASE_URL`、`AGENT_KEY` 等配置。

## 7. 常用运维命令

查看日志：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml logs -f backend
```

重启后端：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml restart backend
```

升级系统：

```bash
git pull
docker compose --env-file .env.production -f docker-compose.prod.yml up -d --build
```

回滚系统镜像：

```bash
# 推荐企业交付时使用 Git tag 标记稳定版本
# 例如：git checkout v1.0.0 && docker compose ... up -d --build
```

## 8. 验收检查

- 可以访问 `https://你的域名`。
- 初始管理员可以登录。
- `/actuator/health` 返回 `UP`。
- 监控目标能生成 Agent Key。
- 目标机器 Agent 心跳成功，CPU/内存/磁盘/网络/进程数有数据。
- 事件工作台可以看到告警。
- AI 专家可以从事件发起调查。
- 审计日志可查看关键操作。
