# Phase 1 迁移与联调说明

> 日期：2026-04-27

## 1. 数据库迁移

执行：

```bash
mysql -uroot -proot -h127.0.0.1 -P3307 aiops_monitor < mysql/migrations/2026-04-27_phase1_target_agent.sql
```

作用：

1. 创建或补齐 `monitor_targets` 的运行字段（enabled/status/heartbeat/ip/version）
2. 补齐 `system_metrics_history.user_id/target_id`
3. 补齐 `incident_log.user_id/target_id`

## 2. 联调顺序（推荐）

1. 登录系统创建监控目标（`/targets` 页面）
2. 复制 `agentKey`
3. 启动模拟 Agent：`tools/agent-simulator/simulate-agent.mjs`
4. 打开 Dashboard 观察实时 CPU/MEMORY 推送
5. 回到目标页确认状态变为 `ONLINE`
6. 停止模拟 Agent，约 90 秒后状态自动变 `OFFLINE`

## 3. 注意事项

1. 轮换 key 后旧 key 立刻失效，Agent 必须更新配置。
2. 禁用目标后 Agent 上报会返回 `403`。
3. 默认离线阈值为 `monitor.target.offline-seconds=90`，可在 `application.yml` 调整。
