# Phase 1 迁移与联调说明

> 日期：2026-04-27

## 1. 数据库迁移

执行：

```bash
mysql -uroot -proot -h127.0.0.1 -P3307 aiops_monitor < mysql/migrations/2026-04-27_phase1_target_agent.sql
mysql -uroot -proot -h127.0.0.1 -P3307 aiops_monitor < mysql/migrations/2026-04-27_phase1_metric_extension.sql
mysql -uroot -proot -h127.0.0.1 -P3307 aiops_monitor < mysql/migrations/2026-04-27_phase1_incident_thresholds.sql
mysql -uroot -proot -h127.0.0.1 -P3307 aiops_monitor < mysql/migrations/2026-04-27_phase1_notification_channels.sql
mysql -uroot -proot -h127.0.0.1 -P3307 aiops_monitor < mysql/migrations/2026-04-27_phase1_alert_noise_controls.sql
```

作用：

1. 创建或补齐 `monitor_targets` 的运行字段（enabled/status/heartbeat/ip/version）
2. 补齐 `system_metrics_history.user_id/target_id`
3. 扩展 `system_metrics_history.disk_usage/net_rx_bytes_per_sec/net_tx_bytes_per_sec/process_count`
4. 补齐 `incident_log.user_id/target_id`
5. 新增 `alert_threshold_config`（按用户阈值配置）
6. 扩展 `incident_log.status/acknowledged_at/resolved_at`
7. 新增 `notification_channel`（通知通道配置）
8. 新增 `notification_delivery_log`（通知投递审计）
9. 扩展 `alert_threshold_config.consecutive_breach_count/silence_seconds`（告警降噪）

## 2. 联调顺序（推荐）

1. 登录系统创建监控目标（`/targets` 页面）
2. 复制 `agentKey`
3. 启动模拟 Agent：`tools/agent-simulator/simulate-agent.mjs`
4. 打开 Dashboard 观察实时 CPU/MEMORY 推送
5. 打开「通知通道」页配置至少 1 条 Webhook 通道（`/settings/notifications`）
6. 回到目标页确认状态变为 `ONLINE`
7. 人为压高阈值触发告警后，在「告警中心」查看并处理事件（`/incidents`）
8. 在告警事件内打开“投递记录”确认通知已发送
9. 停止模拟 Agent，约 90 秒后状态自动变 `OFFLINE`

## 3. 注意事项

1. 轮换 key 后旧 key 立刻失效，Agent 必须更新配置。
2. 禁用目标后 Agent 上报会返回 `403`。
3. 默认离线阈值为 `monitor.target.offline-seconds=90`，可在 `application.yml` 调整。
4. 当前默认告警阈值：`cpu=70`、`memory=90`、`disk=85`、`process-count=400`。
5. 当前默认降噪策略：`consecutive-breach-count=2`、`silence-seconds=180`。
