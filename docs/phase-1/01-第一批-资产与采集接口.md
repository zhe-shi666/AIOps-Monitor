# Phase 1 第一批：资产与采集接口

> 日期：2026-04-27

## 1. 用户侧接口（需 JWT）

### 1.1 创建监控目标
- `POST /api/targets`

请求示例：
```json
{
  "name": "北京-DB-01",
  "hostname": "db-01",
  "description": "核心数据库主机"
}
```

### 1.2 查询监控目标列表
- `GET /api/targets`

### 1.3 查询单个监控目标
- `GET /api/targets/{id}`

### 1.4 更新监控目标
- `PUT /api/targets/{id}`

请求示例：
```json
{
  "name": "北京-DB-01",
  "hostname": "db-01",
  "description": "核心数据库主机（已迁移）",
  "enabled": true
}
```

### 1.5 轮换 Agent Key
- `POST /api/targets/{id}/rotate-key`

### 1.6 删除监控目标
- `DELETE /api/targets/{id}`

## 2. Agent 侧接口（免 JWT，使用 agentKey）

### 2.1 Agent 注册
- `POST /api/agent/register`

请求示例：
```json
{
  "agentKey": "<target-agent-key>",
  "hostname": "db-01",
  "ip": "10.0.0.12",
  "agentVersion": "1.0.0"
}
```

### 2.2 心跳上报
- `POST /api/agent/heartbeat`

请求示例：
```json
{
  "agentKey": "<target-agent-key>",
  "hostname": "db-01",
  "cpuUsage": 36.2,
  "memUsage": 68.4,
  "diskUsage": 57.9,
  "netRxBytesPerSec": 1234567,
  "netTxBytesPerSec": 845210,
  "processCount": 196,
  "timestamp": 1777276800000,
  "ip": "10.0.0.12",
  "agentVersion": "1.0.0"
}
```

## 3. 本批实现行为说明

1. 心跳会写入 `system_metrics_history`，并填充 `user_id`、`target_id`。
2. 心跳会推送 `CPU/MEMORY/DISK/NET_RX/NET_TX/PROCESS_COUNT` 到 `/topic/metrics`，保持大盘实时能力。
3. 目标状态在注册/心跳时置为 `ONLINE`。
4. 若超过 `monitor.target.offline-seconds`（默认 90 秒）无心跳，自动置为 `OFFLINE`。
5. 若目标被禁用（`enabled=false`），Agent 上报会被拒绝。
