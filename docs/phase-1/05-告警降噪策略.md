# Phase 1：告警降噪策略（连续触发 + 静默窗口）

> 日期：2026-04-27

## 1. 策略目标

1. 避免瞬时抖动导致误报。
2. 避免同一问题短时间内重复刷屏。

## 2. 生效规则

1. 当指标值超过阈值时，仅累计“连续超阈值次数”。
2. 达到 `consecutiveBreachCount` 才会创建告警并触发通知。
3. 告警触发后，同目标同指标进入 `silenceSeconds` 静默窗口。
4. 指标恢复到阈值内会清空连续计数。

## 3. 配置入口

1. 页面：`/settings/thresholds`
2. 接口：`GET/PUT /api/settings/thresholds`

配置字段：
1. `consecutiveBreachCount`：连续超阈值次数（1~20）。
2. `silenceSeconds`：静默窗口秒数（10~86400）。

## 4. 默认值

1. `consecutiveBreachCount = 2`
2. `silenceSeconds = 180`
