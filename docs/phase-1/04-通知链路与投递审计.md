# Phase 1：通知链路与投递审计

> 日期：2026-04-27

## 1. 通知通道管理（需 JWT）

### 1.1 查询通道列表
- `GET /api/settings/notification-channels`

### 1.2 新增通道
- `POST /api/settings/notification-channels`

请求示例：
```json
{
  "name": "值班 webhook",
  "type": "WEBHOOK",
  "webhookUrl": "https://example.com/hooks/aiops",
  "secret": "optional-token",
  "enabled": true
}
```

### 1.3 更新通道
- `PUT /api/settings/notification-channels/{id}`

### 1.4 启停通道
- `PUT /api/settings/notification-channels/{id}/enabled`

请求示例：
```json
{
  "enabled": false
}
```

### 1.5 删除通道
- `DELETE /api/settings/notification-channels/{id}`

## 2. 告警投递记录（需 JWT）

### 2.1 查询某条告警的投递记录
- `GET /api/incidents/{id}/deliveries?page=0&size=20`

返回数据包含：
1. 通道名称、目标地址
2. 投递结果（`SUCCESS/FAILED`）
3. HTTP 状态码
4. 错误信息与时间

## 3. 触发与发送规则

1. Agent 上报超过阈值并创建 `OPEN` 告警时，异步触发通知投递。
2. 告警状态变更（`OPEN/ACKNOWLEDGED/RESOLVED`）时，异步发送状态变更通知。
3. 仅对当前用户已启用通道发送。
4. 每次投递都会写入 `notification_delivery_log` 形成审计链路。
5. 删除通知通道不会清除历史投递记录（审计保留）。

## 4. 前端入口

1. 通知通道页面：`/settings/notifications`
2. 告警中心页面：`/incidents`（支持查看每条事件的投递记录）
