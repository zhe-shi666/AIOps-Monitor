# Phase 1：告警升级策略与节奏

> 日期：2026-04-27

## 1. 升级策略配置（需 JWT）

### 1.1 获取策略
- `GET /api/settings/escalation-policy`

返回示例：
```json
{
  "p1Intervals": "1,3,5,10",
  "p2Intervals": "5,15,30",
  "p3Intervals": "15,30,60",
  "updatedAt": "2026-04-27T18:10:00"
}
```

### 1.2 更新策略
- `PUT /api/settings/escalation-policy`

请求示例：
```json
{
  "p1Intervals": "1,2,5",
  "p2Intervals": "5,10,20",
  "p3Intervals": "15,30,60"
}
```

说明：
1. 以上字段单位为分钟，使用逗号分隔整数。
2. 策略按当前登录用户隔离生效。

## 2. 升级触发规则

1. 告警创建后按严重级别分配 `severity`（`P1/P2/P3`）。
2. 系统根据级别读取对应节奏，计算 `nextNotifyAt`。
3. 到达 `nextNotifyAt` 后触发升级通知并递增 `escalationLevel`。
4. 若该级别节奏已耗尽，`nextNotifyAt` 置空，不再继续升级。
5. 仅 `OPEN` 状态的告警参与升级；`ACKNOWLEDGED/RESOLVED` 会停止升级。

## 3. 前端入口

1. 升级策略页面：`/settings/escalation`
2. 告警中心页面：`/incidents`（显示 `severity/escalationLevel/nextNotifyAt`）
