# Phase 1：阈值配置与告警中心接口

> 日期：2026-04-27

## 1. 阈值配置（需 JWT）

### 1.1 获取当前用户阈值
- `GET /api/settings/thresholds`

返回示例：
```json
{
  "cpuThreshold": 70,
  "memoryThreshold": 90,
  "diskThreshold": 85,
  "processCountThreshold": 400,
  "updatedAt": "2026-04-27T17:35:00"
}
```

### 1.2 更新阈值
- `PUT /api/settings/thresholds`

请求示例：
```json
{
  "cpuThreshold": 75,
  "memoryThreshold": 92,
  "diskThreshold": 88,
  "processCountThreshold": 500
}
```

说明：
1. 阈值按“当前登录用户”生效。
2. Agent 心跳告警实时使用该用户阈值。

## 2. 告警中心（需 JWT）

### 2.1 查询告警列表
- `GET /api/incidents?page=0&size=20&status=OPEN&metricName=CPU&hostname=db&keyword=超阈值`

支持过滤参数：
1. `status`：`OPEN/ACKNOWLEDGED/RESOLVED`
2. `metricName`：`CPU/MEMORY/DISK/PROCESS_COUNT`
3. `hostname`：模糊匹配
4. `keyword`：消息内容模糊匹配

### 2.2 更新告警状态
- `PUT /api/incidents/{id}/status`

请求示例：
```json
{
  "status": "ACKNOWLEDGED"
}
```

支持状态：
1. `OPEN`
2. `ACKNOWLEDGED`
3. `RESOLVED`

## 3. 状态流转规则

1. `OPEN -> ACKNOWLEDGED`：记录 `acknowledged_at`。
2. `ACKNOWLEDGED -> RESOLVED`：记录 `resolved_at`。
3. `RESOLVED -> OPEN`：清空 `acknowledged_at/resolved_at`。
