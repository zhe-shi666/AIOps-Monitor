# Phase 1：阈值配置与告警中心接口

> 日期：2026-04-27

## 1. 阈值配置（需 JWT）

### 1.1 获取当前用户阈值
- `GET /api/settings/thresholds`

返回示例：
```json
{
  "cpuThreshold": 70,
  "memoryThreshold": 90,
  "diskThreshold": 85,
  "processCountThreshold": 400,
  "consecutiveBreachCount": 2,
  "silenceSeconds": 180,
  "updatedAt": "2026-04-27T17:35:00"
}
```

### 1.2 更新阈值
- `PUT /api/settings/thresholds`

请求示例：
```json
{
  "cpuThreshold": 75,
  "memoryThreshold": 92,
  "diskThreshold": 88,
  "processCountThreshold": 500,
  "consecutiveBreachCount": 3,
  "silenceSeconds": 300
}
```

说明：
1. 阈值按“当前登录用户”生效。
2. Agent 心跳告警实时使用该用户阈值。
3. `consecutiveBreachCount` 表示连续超阈值多少次后才触发告警。
4. `silenceSeconds` 表示同一目标同一指标触发后进入静默窗口，窗口内不重复告警。

## 2. 告警中心（需 JWT）

### 2.1 查询告警列表
- `GET /api/incidents?page=0&size=20&status=OPEN&metricName=CPU&hostname=db&keyword=超阈值`

支持过滤参数：
1. `status`：`OPEN/ACKNOWLEDGED/RESOLVED`
2. `metricName`：`CPU/MEMORY/DISK/PROCESS_COUNT`
3. `hostname`：模糊匹配
4. `keyword`：消息内容模糊匹配

返回关键字段（新增）：
1. `severity`：`P1/P2/P3`
2. `escalationLevel`：当前升级层级
3. `nextNotifyAt`：下一次计划通知时间（OPEN 状态）

### 2.2 更新告警状态
- `PUT /api/incidents/{id}/status`

请求示例：
```json
{
  "status": "ACKNOWLEDGED"
}
```

返回关键字段（新增）：
1. `severity`
2. `escalationLevel`
3. `nextNotifyAt`

支持状态：
1. `OPEN`
2. `ACKNOWLEDGED`
3. `RESOLVED`

## 3. 状态流转规则

1. `OPEN -> ACKNOWLEDGED`：记录 `acknowledged_at`。
2. `ACKNOWLEDGED -> RESOLVED`：记录 `resolved_at`。
3. `RESOLVED -> OPEN`：清空 `acknowledged_at/resolved_at`。
4. 非 `OPEN` 状态会暂停升级通知（清空 `next_notify_at`）。
5. 重新 `OPEN` 会重置升级层级并按策略重新计算下一次通知时间。
